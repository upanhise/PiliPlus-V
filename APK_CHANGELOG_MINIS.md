# PiliPlus-V 本次 APK 修改说明

构建分支：`minis/dev-start`

## 下载建议

Android 手机一般下载：

- `Android_arm64-v8a`

## 本次主要改动

### 1. 更新检查改为 fork 仓库

应用内检查更新现在指向：

- `upanhise/PiliPlus-V`

不再读取上游仓库 release。

### 2. 更新弹窗支持忽略版本

自动检查更新时，如果不想升级当前版本，可以点击：

- `忽略此版本`

忽略后：

- 自动检查不再提示该版本
- 手动检查仍可看到更新信息

### 3. 修复误报更新

优化版本比较逻辑，支持识别：

- `2.0.9`
- `2.0.9.2`
- `v2.0.9.2`
- `2.0.9+5051`
- `2.0.9-commit`
- `2.0.9.2-minis.1`

避免已经是最新版本但仍弹更新。

### 4. 视频简介提示样式增强

视频简介中的：

- 争议信息
- AI 生成内容提示

改为更醒目的红色加粗样式。

### 5. 新增自定义番剧源设置骨架

设置页新增：

- 启用自定义番剧源
- 自定义番剧源地址
- 官方权限失败时尝试自定义番剧源
- 显示当前番剧源提示

当前状态：

- 只提供配置入口
- 实际自定义源接口仍在开发中
- 不内置任何第三方源

### 6. 番剧播放源策略加固

已加入：

- 播放前刷新 VIP 状态
- 5 分钟 VIP 刷新节流
- VIP / 未登录 / 未知状态区分
- 大会员不自动降级
- 未知 VIP 状态保守处理，不 fallback
- 番剧源只有在官方源/自定义源发生切换时才提示

### 7. 自定义番剧源 URL 校验

保存源地址时会校验：

- 只允许 `http://` 或 `https://`
- 必须有 host
- 自动移除尾部 `/`

非法地址不会保存。

### 8. Fork CI 构建修复

CI 已修复并成功产出 APK：

- `flutter analyze --no-fatal-infos`
- 自动生成 `pili_release.json`
- 构建 arm64-v8a / armeabi-v7a / x86_64
- artifact 缺失时直接失败
